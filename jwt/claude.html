Understanding JSON Web Tokens step by step

1
JWT Structure

A JWT consists of three parts separated by dots (.)

Header

Algorithm & Token Type

Payload

Claims & Data

Signature

Verification Hash

2
Creating the Header
{ "alg": "HS256", "typ": "JWT" }

The header is Base64URL encoded:

eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9
3
Creating the Payload
{ "sub": "1234567890", "name": "John Doe", "iat": 1516239022, "exp": 1516242622 }

The payload is also Base64URL encoded:

eyJzdWIiOiIxMjM0NTY3ODkwIiwibmFtZSI6IkpvaG4gRG9lIiwiaWF0IjoxNTE2MjM5MDIyLCJleHAiOjE1MTYyNDI2MjJ9
4
Creating the Signature
HMACSHA256( base64UrlEncode(header) + "." + base64UrlEncode(payload), secret )

The signature ensures the token hasn't been tampered with:

SflKxwRJSMeKKF2QT4fwpMeJf36POk6yJV_adQssw5c
5
Complete JWT Token

The final JWT is created by joining all three parts with dots:

eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJzdWIiOiIxMjM0NTY3ODkwIiwibmFtZSI6IkpvaG4gRG9lIiwiaWF0IjoxNTE2MjM5MDIyLCJleHAiOjE1MTYyNDI2MjJ9.SflKxwRJSMeKKF2QT4fwpMeJf36POk6yJV_adQssw5c

  3. Recreate signature using the secret

  6. Validate claims
