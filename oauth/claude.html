1

User initiates authorization

User clicks "Login with OAuth Provider" in the client application. The client redirects the user to the authorization server with parameters including client_id, redirect_uri, response_type=code, and scope.

2

User authenticates and authorizes

The authorization server presents a login page to the user. After successful authentication, the user is shown the permissions being requested and chooses to grant or deny access.

3

Authorization code returned

If the user grants permission, the authorization server redirects back to the client application with an authorization code as a query parameter.

https://clientapp.com/callback?code=abc123&state=xyz
4

Exchange code for access token

The client application makes a server-to-server request to exchange the authorization code for an access token, including client credentials for authentication.

POST /token client_id=123&client_secret=secret&code=abc123&grant_type=authorization_code
5

Access token issued

The authorization server validates the request and issues an access token (and optionally a refresh token) to the client application.

Access Token: eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...
Token Type: Bearer
Expires In: 3600 seconds
6

Access protected resources

The client application can now use the access token to make authenticated requests to the resource server API on behalf of the user.

GET /api/user/profile Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...
