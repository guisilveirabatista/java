JVM Internal Architecture

Class Loader Subsystem
Loading: Finds and loads .class files

Linking:
• Verification: Ensures bytecode is valid
• Preparation: Allocates memory for static variables
• Resolution: Replaces symbolic references

Initialization: Executes static initializers
Method Area
• Class metadata
• Static variables
• Method bytecode
• Constant pool
Heap Memory
• Object instances
• Instance variables
• Arrays
• Garbage collected
Stack Memory
• Method call frames
• Local variables
• Partial results
• Thread-specific
Execution Engine
Interpreter: Executes bytecode line by line

Just-In-Time (JIT) Compiler: Compiles frequently used bytecode to native machine code for better performance

Garbage Collector: Automatically manages memory by removing unused objects from heap
